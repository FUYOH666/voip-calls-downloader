| Версия | Поддерживается |
| ------ | -------------- |
| 1.x    | :white_check_mark: |

## Сообщение об уязвимостях

Если вы обнаружили уязвимость безопасности, пожалуйста, **не создавайте публичный issue**. 

Вместо этого отправьте email на: [security@scanovich.ai](mailto:security@scanovich.ai)

Включите следующую информацию:
- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное влияние
- Предложения по исправлению (если есть)

Мы постараемся ответить в течение 48 часов и предоставить обновление о прогрессе исправления.

## Что считать уязвимостью

Уязвимости безопасности включают:
- Утечка секретов или учетных данных
- Небезопасная обработка пользовательских данных
- Уязвимости, позволяющие выполнение произвольного кода
- Небезопасное хранение паролей или токенов
- Проблемы с аутентификацией и авторизацией

## Безопасное использование

### Хранение секретов

- **Никогда** не коммитьте `.env` файлы в git
- Используйте `.env.example` как шаблон
- Храните секреты только в переменных окружения
- Не логируйте пароли, токены или другие секреты

### Рекомендации

1. Регулярно обновляйте зависимости
2. Используйте сильные пароли для учетных записей
3. Ограничьте доступ к `.env` файлам (права доступа 600)
4. Не используйте один и тот же пароль для разных сервисов
